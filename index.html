Sat, Jan 15, 2022 记录使用hugo搭建blog的坑 github新建repository时不要添加README file 如果添加了README file，需要把default branch由main改成master，然后在github上把README file删了，否则访问https://coffeentv.github.io/时README file会覆盖原来的内容 在hugo上新建md文件时，把draft一行去掉，此时不要用notepad++这个神坑软件编辑，否则会把你的md文件变成.md.bak文件。建议使用typora 使用git push -u origin master如果报错，则先运行git pull –rebase origin master，再push。参考https://blog.csdn.net/qq_44503987/article/details/107702455 命令行 控制台 hugo new site blog cd blog hugo new welcome.md hugo new post/paper1.md cd themes git clone http://github.com/spf13/hyde.git cd .. hugo server –theme=hyde –buildDrafts –watch hugo –theme=hyde –baseUrl=“https://coffeeNtv.github.io/" cd public git init git remote add origin https://github.com/coffeeNtv/coffeeNtv.github.io.git git add -A git commit -m “first commit” git pull –rebase origin master git push -u origin master
